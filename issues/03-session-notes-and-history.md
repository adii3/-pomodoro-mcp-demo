## What to build

Connect the real API to the user-facing history flow. A completed session should let the user record a short note and then appear in the recent session list with its key details.

## Acceptance criteria

- [ ] Completing a session supports an optional note
- [ ] Saving a session sends the real API request
- [ ] Recent sessions are loaded from the backend
- [ ] The UI renders recent sessions with phase, duration, time, and note
- [ ] New sessions appear in history after successful save
